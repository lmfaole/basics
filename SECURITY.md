# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Report vulnerabilities privately to `olejorgenbakken@gmail.com`. Include:

- the affected package version
- the impacted component or API
- reproduction details or a proof of concept
- any suggested mitigation if you already have one

If a secret has been committed accidentally, rotate it first and then report the exposure so the repository history and release process can be reviewed.

I will acknowledge receipt as soon as practical and follow up with status updates during triage and remediation.
